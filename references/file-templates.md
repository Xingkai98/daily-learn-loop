# 文件模板约定

每个材料文件必须有稳定结构，不得缺少以下必备块。允许根据主题增加小节。

## 1. lesson.md

```
# <keyword>
## 1. 定义 / 背景
## 2. 核心原理 / 架构 / 机制
## 3. 关键流程 / 算法 / 设计步骤
## 4. 实际应用场景
## 5. 常见误区
## 6. 面试关注点 / 工程取舍
## 来源
```

## 2. exam.md

```
# <keyword> — Exam
（开头：题目设计依据，标注来源）
## N. <题目名>（⭐/⭐⭐/⭐⭐⭐）
（只放题目不放答案，标注题型依据或知识点来源）
## 来源
```

## 3. exam-answers.md

```
# <keyword> — Exam Answers
（开头：答案依据，标注来源）
## N. <题目名>（难度）
（标准答案、解析、常见错误/扣分点、主要依据）
## 来源
```

## 4. interview.md

```
# <keyword> — Mock Interview
（开头：面试题设计依据，标注来源）
## N. <题目名>
（题目正文 + 追问：列表呈现 + 题型依据）
## 来源
```

## 5. interview-answers.md（核心复习文档）

```
# <keyword> — Interview Reference Answers
（开头：评分标准依据，标注来源）
## N. <题目名>
- 题目原文：（原样写入）
- 追问：（原样写入）
- 满分回答要点：（评分 checklist）
- 示范回答（主问题）：（普通段落/列表，不用代码块）
- 示范回答（追问）：（逐条追问 N 标注）
- 详细解析：（考察点、面试官意图、深度预期）
- 常见错误：（典型扣分回答及纠正）
- 主要依据：（引用来源）
## 来源
```

## 6. interview-transcript.md

```
# <keyword> — Interview Transcript
（日期、主题、状态）
（每题：问题、用户原始回答、点评、追问过程）
（文末：总体评价 — 优点、待加强、结论）
## 来源
```

**用户回答整理规则：** 用户原始回答可能来自语音输入，口语化严重、逻辑跳跃。归档时保留原意的前提下整理为结构化表述——合并重复观点、去口头禅和填充词、理顺因果和总分关系。但答案内容原样保留（即使错误，在点评中纠正）。
